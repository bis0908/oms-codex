# UI 검증 수단 폴백

## 목적

특정 UI 제어 도구의 설치·연결 상태와 검증 품질을 분리한다. 선호 도구가
없어도 가능한 검증을 계속하되, 요구사항보다 약한 증거로 구현 완료를
승인하지 않는다.

## 실행 모드

| 모드 | 목표 | 도구 한계 처리 |
|---|---|---|
| `completion` | 구현 완료와 상태 전이를 증명 | 모든 필수 기준에 등가 증거가 없으면 완료 금지 |
| `audit` | 구현·미구현·미확인 항목을 전수 분류 | 가능한 게이트를 끝까지 실행하고 `수정필요`·검증 부채로 보고 |

감사 모드의 `completed`는 감사 보고서가 완성됐다는 뜻이지 구현이 완료됐다는
뜻이 아니다. tracker 완료 전이와 커밋은 evaluator approved와 사용자 검증을
별도로 요구한다.

## 수단 선택 순서

1. 사용자가 지정했거나 현재 연결된 UI 제어 도구를 한 번 확인한다.
2. 사용 가능한 다른 브라우저·GUI 자동화 또는 프로젝트의 기존 E2E runner를
   사용한다.
3. 실제 production 컴포넌트를 import하는 컴포넌트 런타임 테스트로 클릭,
   입력, 제출, 상태 전환, 포커스, ARIA 변화를 검증한다.
4. HTTP 스모크, API·통합 테스트, build 결과와 정적 라우트·핸들러 추적을
   조합한다.
5. 픽셀·overflow처럼 자동 대체가 불가능한 시각 항목은 재현 절차를 제시하고
   사용자 관찰 또는 사용자가 제공한 스크린샷으로 확인한다.

같은 요청에서 가용성 실패가 확인된 도구를 반복 호출하지 않는다. UI 도구
실패는 다른 수단으로 전환할 신호이며, 그 자체를 agent 실행 실패로 세지
않는다. 새 production dependency를 폴백 목적으로 추가하지 않는다.

## 증거 등가표

| 요구사항 | 승인 가능한 증거 | 단독으로 부족한 증거 |
|---|---|---|
| 페이지 로드·핵심 DOM | 브라우저/E2E 또는 HTTP 2xx + production 컴포넌트 렌더 테스트 | 파일 존재, JSX 문자열 검색 |
| 버튼·폼·상태 전환 | UI 자동화 또는 production 컴포넌트의 사용자 이벤트 테스트 | handler 이름 존재, 스냅샷만 존재 |
| 라우팅·API 호출 | UI/E2E 또는 이벤트 테스트 + 실제 route/API 통합 테스트 | `href`·`fetch` 문자열만 존재 |
| 모달·포커스·키보드 | UI 자동화 또는 `activeElement`, ESC, ARIA를 검증하는 컴포넌트 테스트 | CSS·ARIA 속성 정적 존재만 확인 |
| 반응형·겹침·hover 시각 | viewport별 스크린샷/실제 관찰 | 반응형 클래스 존재만 확인 |
| 외부 인증·결제·알림 | 격리된 통합 smoke 또는 사용자 승인 환경의 실제 관찰 | mock 단위 테스트만 통과 |

정적 분석은 누락된 handler, 존재하지 않는 route, 호출되지 않는 API 같은
부정 증거를 확정하는 데 사용할 수 있다. 반대로 코드 모양만으로 실제 동작을
긍정 판정하지 않는다.

## 판정

- 모든 필수 기준이 등가 증거로 확인되면 사용한 수단과 무관하게 approved를
  허용한다.
- 코드 결함이 확인되면 사용하지 못한 상위 도구와 무관하게 `수정필요`로
  판정한다.
- 감사 모드에서 일부 기준이 미확인이면 해당 항목을 `?`와 검증 부채로 남기고
  보고서를 완성한다. 후속 진단 게이트는 계속 실행한다.
- completion 모드에서 필수 기준이 미확인이면 완료 전이와 커밋을 금지하고
  `needs-input` 또는 실제 환경 차단일 때만 `blocked`로 판정한다.
- auth, destructive, payment처럼 안전상 실제 환경 검증이 필요한 기준은 약한
  폴백만으로 승인하지 않는다.

## 보고 필드

UI 검증 보고서에는 다음을 남긴다.

```text
UI 검증 모드: <completion | audit>
선호 UI 검증 수단: <도구 | 자동 선택>
실제 UI 검증 수단: <도구·테스트·HTTP·사용자 관찰 조합>
대체 사유: <없음 | 선호 도구 실패 사유>
증거 커버리지: <요구사항별 증거 경로·명령>
미검증 항목: <없음 | 항목·이유·해소 조건>
```
